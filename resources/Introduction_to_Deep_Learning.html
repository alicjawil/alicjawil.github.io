The Rectified Linear Activation Function¶

Activation Functions are functions applied at each node to convert nodes input into output.

ReLU function - shown to lead to very high performance networks.

The functions take a sigle number as input, if number is positive it returns the input number, if number is negative it returns 0.

Example:

  • relu(3) = 3
  • relu(-5) = 0

Weights in NN are set during the model trainng process to optimize predictive accuracy.

# Define predict_with_network()
def predict_with_network(input_data_row, weights):

    # Calculate node 0 value
    node_0_input = (input_data_row * weights['node_0']).sum()
    node_0_output = relu(node_0_input)

    # Calculate node 1 value
    node_1_input = (input_data_row * weights['node_1']).sum()
    node_1_output = relu(node_1_input)

    # Put node values into array: hidden_layer_outputs
    hidden_layer_outputs = np.array([node_0_output, node_1_output])
    
    # Calculate model output
    input_to_final_layer = (hidden_layer_outputs * weights['output']).sum()
    model_output = relu(input_to_final_layer)
    
    # Return model output
    return(model_output)

Big breakthrough in deep learning is using multiple hidden layers as opposed to only one hidden layer.

Many hidden layers:

  • use values from a previous hidden layers to fill in values in the next hidden layer. We use activation function ReLU at each node.
  • a few years ago state of the art was 15 hidden layers but now it can easily scale out to 1000 hidden layers

Deep Learning is a Representation Learning :

  • deep network internally builds representations of patterns in the data
  • it partially removes the need for feature engineering

In Deep Learning the modeler does not need to specify the interactions or tell the model what to look for. Training model gets weights to find relevant patterns in the data to make better predictions.

The last layer of the interaction captures the most high - level interactions.

Gradient Descent¶

  • finding the most optimized loss function.
  • going opposite the slope means moving to lower numbers
  • too big steps might make us miss the optimized point - that's why we use Learning Rate.
    • update each weight by substracting learning rate * slope

Backpropagation¶

Backpragation takes the output layer error and propagates it backwords through the hidden layers towards the input layer.

  • frirst, gradient descent updates all weights in neural network
  • comes from chain rule of calculus
  • complex math but what's more important - understand the process bc you will use library to do the math(duh)
  • always do forward propagation first before we do backward propagation

Backpropagation process¶

  • Go back one layer at a time

  • Gradients for weight is product of:

    1. Node value feeding into that weight
    2. Slope of loss function w.r.t node it feeds into
    3. Slope of activation function at the node it feeds into

Backpropagation: Recap¶

  1. Start at some random set of weights
  2. Use forward propagation to make a prediction
  3. Use backward propagation to calculate the slope of the loss function w.r.t each weight
  4. Multiply that slope by the learning rate, and subtract from the current weights
  5. Keep going with that cycle until we get to a flat part

Stochastic gradient descent¶

  • It is common to calculate slopes on only a subset of the data (‘batch’)
  • Use a different batch of data to calculate the next update
  • Start over from the beginning once all data is used Each time through the training data is called an epoch
  • When slopes are calculated on one batch at a time: stochastic gradient descent

Optimization¶

Stochastic Gradient Descent is a good choice ( what about the Mini Batch Gradient Descent)

Common challenge for relu activation function: the dying neuron problem : that happens when neuron takes a value of less than 0 for all rows of data

'Dead' Neuron - once node starts negative, it might continue only getting negative inputs. It contributes nothing to the model.

Common challenge for tanh activation function : 'Vanishin Gradients' - occurs when many layers have very small slopes, due to being on the flat part of the tanh curve.

Diagnosing optimization problems Which of the following could prevent a model from showing an improved loss in its first few epochs?

Learning rate too low. Learning rate too high. Poor choice of activation function.

Model Validation¶

In Deep Learning k-fold is not a good option for validation. Repeated cross validation would take a long time.

In Keras we just use validation_split. Our Goal:

  • keep trainig while validation scores are improving and stop training once validation scores are not improving anymore.
  • to do so we use EarlyStopping monitor, and set in up in .fit as a list. Patience is how many epochs we tolerate to be disimproving.

Model Capacity is key consideration for deep learning models

Model Capacity is ability to capture predictive patterns in data. Increasing numbers of nodes, layers etc increases capacity.

Workflow for optimizing model capacity:

  1. Start with small network
  2. Get the validation score
  3. Keep adding capacity as long as the score is improving
  4. Experiment
